Browser-Based Deployment, Zero Setup

Deploy OpenClaw using the Nebius web console — no Docker, no CLI, no local setup. Point, click, deploy. This workshop walks you through creating a Nebius account, launching a compute instance with OpenClaw pre-configured, connecting Token Factory for inference, and accessing the Control UI — all from your browser.

Who It's For
Anyone who wants a running OpenClaw deployment without touching a terminal or installing anything locally
What You Get
A cloud-hosted OpenClaw instance on Nebius with Token Factory inference, accessible via browser
Time Required
~20 minutes from account creation to a live, accessible agent

What You'll Build

  • A cloud-hosted OpenClaw instance running on Nebius compute
  • Connected to Token Factory for GPU-powered inference with no local GPU
  • Accessible via the Control UI from any browser, anywhere

What We'll Cover

  • Creating a Nebius account — sign-up, billing setup, and navigating the console
  • Navigating to Compute — finding the right service, choosing a region, selecting an instance type
  • Launching an instance with OpenClaw — using the marketplace image or startup script
  • Configuring Token Factory — setting API keys and model selection via the console UI
  • Accessing the Control UI — connecting to your instance's public IP and authenticating
  • Security basics — firewall rules, token authentication, and HTTPS considerations

Prerequisites

  • A web browser (Chrome, Firefox, Safari, or Edge)
  • A valid email address for Nebius account creation
  • A payment method for Nebius billing (free tier available)

Step-by-Step Guide

1
Create a Nebius Account 5 min

Sign up for Nebius and set up billing so you can launch compute resources.

  1. Go to console.nebius.com and click Sign Up
  2. Verify your email address and complete the registration form
  3. Navigate to Billing and add a payment method (free tier credits apply)
Tip: Nebius offers free trial credits for new accounts. You can run a small compute instance for several hours at no cost while you explore.
Checkpoint: You can log into the Nebius console and see the main dashboard.
2
Launch a Compute Instance with OpenClaw 5 min

Navigate to the Compute service and launch an instance pre-configured with OpenClaw.

  1. In the Nebius console, navigate to Compute from the left sidebar
  2. Click Create Instance and select a region close to you
  3. Choose a CPU instance type (2 vCPU / 4 GB RAM is sufficient)
  4. Under Image, select the OpenClaw marketplace image or paste the startup script
  5. Configure the firewall to allow inbound traffic on port 18789
  6. Click Create and wait for the instance to reach Running status
Tip: A CPU instance is all you need — inference happens on Token Factory's GPUs, not on your compute instance. This keeps costs low.
Checkpoint: Your instance shows Running status with a public IP address assigned.
3
Configure Token Factory 5 min

Set up Token Factory credentials so your OpenClaw instance can access GPU-powered inference.

  1. Go to tokenfactory.nebius.com and create an API key
  2. In the Nebius console, navigate to your instance's Environment Variables section
  3. Set TOKEN_FACTORY_API_KEY to your API key
  4. Set INFERENCE_MODEL to zai-org/GLM-5
  5. Set OPENCLAW_GATEWAY_TOKEN to a password of your choice
# Environment variables configured via the console UI: TOKEN_FACTORY_API_KEY={your-token-factory-key} TOKEN_FACTORY_URL=https://api.tokenfactory.nebius.com/v1 INFERENCE_MODEL=zai-org/GLM-5 OPENCLAW_GATEWAY_TOKEN={your-password}
Tip: You can change the model at any time by updating the INFERENCE_MODEL variable. Try deepseek-ai/DeepSeek-R1-0528 for reasoning tasks.
Checkpoint: All four environment variables are saved in the console.
4
Access the Control UI 5 min

Open your browser and connect to the running OpenClaw instance.

  1. Copy your instance's public IP from the Nebius console
  2. Open http://{your-ip}:18789 in your browser
  3. Authenticate with the gateway token you set in Step 3
  4. Send a test message to verify the agent responds via Token Factory
# Open in your browser (replace with your instance IP) http://203.0.113.42:18789/#token={your-password}
Tip: For production use, set up a domain name and HTTPS. The Nebius console supports load balancer configuration with TLS termination.
Checkpoint: You see the Control UI dashboard and can send and receive messages from your agent.
