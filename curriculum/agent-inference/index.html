Nebius Serverless + Token Factory — Production Agent Deployment

Deploy OpenClaw on Nebius Serverless (CPU-only) and connect it to Token Factory for inference. No Mac Minis, no local installs, no security headaches — a single serverless endpoint replaces $1,000 in hardware, and you can run 100 agents for the same cost. You'll choose the right open model, configure structured outputs for reliable tool calling, and optimize cost and latency.

Who It's For
Developers ready to deploy production agents without managing GPU infrastructure
Value

Step-by-Step Guide

1
Install the Nebius CLI 5 min

Install and configure the Nebius AI Cloud CLI.

  1. Download and install the Nebius CLI from the official docs
  2. Run the login flow to authenticate with your Nebius account
  3. Verify your project ID is set correctly
# Install Nebius CLI (macOS) curl -sSL https://storage.eu-north1.nebius.cloud/cli/install.sh | bash # Login to your account nebius auth login # Verify your config cat ~/.nebius/config.yaml
Tip: If you don't have a Nebius account yet, sign up at nebius.com — mentors can help you get set up.
Checkpoint: Running nebius iam whoami returns your user info without errors.
2
Get Your Network and Subnet IDs 3 min

Look up the network and subnet IDs you'll need to deploy your serverless endpoint.

  1. List your VPC networks to find the network ID
  2. Get the default subnet ID within that network
  3. Save both IDs — you'll use them in the next step
# List networks nebius vpc network list # Get default subnet ID nebius vpc subnet get-by-name --name default-subnet \ --format jsonpath='{.metadata.id}'
Checkpoint: You have a network ID and subnet ID saved for the next step.
3
Deploy OpenClaw on Serverless 10 min

Create a serverless endpoint running OpenClaw. CPU-only container — no GPU needed for the agent orchestrator.

  1. Generate an auth token for your endpoint
  2. Create the serverless endpoint using the Nebius CLI
  3. Wait ~30 seconds for the endpoint to become active
  4. Retrieve the public IP to test connectivity
# Generate auth token export AUTH_TOKEN=$(openssl rand -hex 32) # Deploy OpenClaw on Serverless nebius msp serverless v1alpha1 endpoint create \ --name openclaw-agent \ --container-image openclaw:latest \ --container-template-resources-platform cpu-d3 \ --container-template-resources-preset 4vcpu-16gb \ --port 8080 \ --username admin \ --password "$AUTH_TOKEN" \ --network-id <your-network-id> \ --parent-id <your-project-id> # Get endpoint ID export ENDPOINT_ID=$(nebius msp serverless v1alpha1 endpoint get-by-name \ --name openclaw-agent --format jsonpath='{.metadata.id}') # Get public IP export ENDPOINT_IP=$(nebius msp serverless v1alpha1 endpoint get $ENDPOINT_ID \ --format jsonpath='{.status.public_endpoints[0]}')
Tip: CPU-only is intentional — OpenClaw is the orchestrator, not the model. Token Factory handles inference on GPUs.
Tip: The 4vcpu-16gb preset is plenty for most agent workloads. Scale up later if needed.
Checkpoint: curl http://$ENDPOINT_IP:8080/health returns a 200 response.
4
Connect to Token Factory 5 min

Get your Token Factory API key and configure OpenClaw to use it for inference.

  1. Go to Token Factory in the Nebius console and create an API key
  2. Set the API key and endpoint URL in your OpenClaw config
  3. Choose a model from the catalog (we recommend starting with Llama 3.1 70B)
  4. Test a basic completion to verify the connection
# Set Token Factory credentials in OpenClaw export TF_API_KEY=<your-token-factory-api-key> # Test the connection directly curl https://api.studio.nebius.com/v1/chat/completions \ -H "Authorization: Bearer $TF_API_KEY" \ -H "Content-Type: application/json" \ -d '{ "model": "meta-llama/Meta-Llama-3.1-70B-Instruct", "messages": [{"role": "user", "content": "Hello, world!"}], "max_tokens": 100 }'
Tip: Token Factory uses an OpenAI-compatible API — if your code works with OpenAI, it works with TF.
Tip: Start with Llama 3.1 70B for a good balance of quality and cost. Try DeepSeek for coding tasks.
Checkpoint: The curl command returns a valid chat completion response from Token Factory.
5
Configure Structured Outputs 10 min

Set up JSON mode and structured outputs so your agent can reliably call tools and parse responses.

  1. Enable JSON mode in your Token Factory requests
  2. Define a tool schema for your use case (e.g., email actions, database queries)
  3. Configure OpenClaw to use structured outputs for tool calling
  4. Test that the model returns valid JSON matching your schema
# Example: structured output for tool calling curl https://api.studio.nebius.com/v1/chat/completions \ -H "Authorization: Bearer $TF_API_KEY" \ -H "Content-Type: application/json" \ -d '{ "model": "meta-llama/Meta-Llama-3.1-70B-Instruct", "messages": [ { "role": "user", "content": "Classify this email: Meeting tomorrow at 3pm" } ], "response_format": {"type": "json_object"}, "max_tokens": 200 }'
Tip: JSON mode forces the model to output valid JSON — no more parsing errors in your agent loop.
Tip: For complex tool schemas, test with a few examples before wiring into the full workflow.
Checkpoint: Your agent returns valid JSON that matches your tool schema on 5 consecutive test runs.
6
Build Your Agent Workflow 15 min

Wire everything together into a real business workflow.

  1. Pick a use case from the examples or bring your own
  2. Configure the agent loop: receive input → call model → execute tool → return result
  3. Test with real-world data (sample emails, documents, etc.)
  4. Iterate on prompts and tool schemas until results are reliable
Tip: Start with a simple workflow and add complexity. A 3-tool agent beats a 10-tool agent that breaks.
Tip: Use the mentor chat for prompt engineering help.
Checkpoint: Your agent handles 3 different inputs correctly end-to-end without manual intervention.
7
Cost Analysis & Production Planning 10 min

Benchmark costs, compare models, and plan your production deployment.

  1. Run your workflow 10 times and note the token usage
  2. Calculate monthly cost at your expected volume
  3. Compare with GPT-4 / Claude API pricing for the same workload
  4. Document your configuration for production deployment
# Check endpoint logs nebius msp serverless v1alpha1 endpoint logs $ENDPOINT_ID # View endpoint status nebius msp serverless v1alpha1 endpoint get $ENDPOINT_ID
Tip: Most agent workflows cost 60–80% less on Token Factory vs. closed APIs.
Tip: Save your CLI commands in a deploy script for easy redeployment.
Checkpoint: You have a working agent, a cost estimate, and a deploy script you can run again anytime.
