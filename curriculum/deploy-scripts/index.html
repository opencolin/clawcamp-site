One-Command Automated Setup

Deploy OpenClaw with a single command using the official install scripts. SSH into a Nebius VM, run one script, and walk away with a fully configured Gateway, Token Factory connection, and accessible Control UI. This is the fastest path to a repeatable, automated cloud deployment.

Who It's For
Developers comfortable with SSH who want a fast, repeatable deployment without manual configuration
What You Get
A fully automated OpenClaw deployment on Nebius with Gateway, Token Factory, and connectivity configured
Time Required
~15 minutes from SSH login to a verified, running deployment

What You'll Build

  • A fully configured Gateway + Token Factory connection with zero manual setup
  • A repeatable process you can run on any new VM in under a minute

What We'll Cover

  • What the script does under the hood — understanding the automation so you can customize it

Prerequisites

  • A Nebius VM with a public IP (see the Web Console workshop to create one)
  • An SSH client (Terminal on macOS/Linux, or PuTTY on Windows)
  • A Token Factory API key (sign up at tokenfactory.nebius.com)

Step-by-Step Guide

1
SSH into Your Nebius VM 3 min

Connect to your Nebius compute instance via SSH.

  1. Open your terminal
  2. SSH into the VM using the public IP from the Nebius console
  3. Verify you have root or sudo access
ssh ubuntu@{your-instance-ip}
# Verify sudo access sudo whoami # Should print: root
Tip: If you created the VM via the Nebius console, your SSH key was configured during instance creation. Use the same key pair.
Checkpoint: You are logged into the VM and see a shell prompt.
2
Run the Install Script 5 min

Execute the one-command install script that sets up everything automatically.

  1. Export your Token Factory API key and gateway token
  2. Run the install script — it handles Node.js, OpenClaw, and gateway configuration
  3. Wait for the script to complete (typically 2–3 minutes)
export TOKEN_FACTORY_API_KEY={your-token-factory-key} export OPENCLAW_GATEWAY_TOKEN={your-password} export INFERENCE_MODEL=zai-org/GLM-5
curl -fsSL https://get.openclaw.dev | bash
Tip: The script is idempotent — you can safely run it again if something fails. It detects what's already installed and skips those steps.
Checkpoint: The script prints OpenClaw gateway started successfully at the end.
3
Verify the Deployment 4 min

Run health checks and inspect logs to confirm everything is working.

  1. Check the gateway process status
  2. Inspect the gateway logs for any errors
  3. Run the built-in health check command
  4. Test connectivity from your local browser
# Check gateway status openclaw status
# View recent logs openclaw logs --tail 20
# Health check curl -s http://localhost:18789/health | jq .
Tip: If the health check fails, check that port 18789 is open in your Nebius firewall rules. The install script configures the local firewall but not the cloud security group.
Checkpoint: Health check returns {"status":"ok"} and the Control UI loads in your browser.
4
Connect and Test 3 min

Open the Control UI and send a message to confirm end-to-end connectivity.

  1. Open http://{your-ip}:18789 in your browser
  2. Authenticate with your gateway token
  3. Send a test message and verify the agent responds
# Open in your browser http://{your-instance-ip}:18789/#token={your-password}
Tip: To change models later, update the INFERENCE_MODEL environment variable and restart the gateway with openclaw gateway restart.
Checkpoint: You received a response from your agent via Token Factory.
