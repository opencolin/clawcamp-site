Calibrate. Teleoperate. Train. Infer. All From One CLI.

Use Solo CLI to work with real robotic hardware — SO-101, Koch, LeKiwi, OpenDroid (Realman R1D2), Unitree G1, and Booster. Calibrate arms, teleoperate, record training datasets, and train vision-language-action (VLA) models like ACT, Pi0, SmolVLA, and GROOT N1.5 on Nebius GPUs. Then deploy your trained policy behind Nebius Serverless + Token Factory for cloud-based inference. Motors are already set up — you start with calibration and go.

Step-by-Step Guide

1
Install Solo CLI & Nebius CLI 5 min

Install Solo CLI for robotics operations and the Nebius CLI for cloud deployment.

  1. Clone and install Solo CLI from source (recommended for hackathons), or install from PyPI
  2. Run the interactive setup to configure your hardware and preferences
  3. Install the Nebius CLI and authenticate
# Install Solo CLI from source (recommended for hackathons) git clone https://github.com/GetSoloTech/solo-cli.git cd solo-cli uv pip install -e . # Or install from PyPI uv pip install solo-cli # Run interactive setup solo setup # Install Nebius CLI curl -sSL https://storage.eu-north1.nebius.cloud/cli/install.sh | bash nebius auth login
Tip: Solo CLI requires Python 3.12+. If you're on an older version, use uv python install 3.12 first.
Tip: The solo setup command saves your config to ~/.solo/config.json — it remembers your hardware and preferences.
Tip: Full Solo CLI docs: Solo CLI README. LeRobot framework (used under the hood): github.com/huggingface/lerobot
Checkpoint: Running solo status shows your system info and nebius iam whoami returns your user info.
2
Calibrate Your Robot 10 min

Connect to your robotic arm and run calibration. Motors are already set up — just calibrate the coordinate system. Available robots: SO-101, Koch, LeKiwi, and more.

  1. Run the calibration command to set workspace boundaries for all joints
  2. Move each joint to its limit when prompted
  3. Test teleoperation — move the follower arm with the leader arm
# Calibrate all joints (motors are already set up) solo robo --calibrate all # Test teleoperation — move the arm with the leader arm solo robo --teleop
Tip: Motors are already set up — skip motor setup and go straight to calibration.
Tip: During calibration, move each joint to its limit when prompted. This sets the workspace boundaries.
Tip: Robot docs: SO-101 | Koch | LeKiwi (assembly) | OpenDroid quick start
Checkpoint: Calibration completes without errors and you can teleoperate the arm smoothly.
3
Record Training Data 15 min

Teleoperate the arm to perform a task and record episodes as training data for your VLA model. Each episode captures the camera feed, joint positions, and your control inputs.

  1. Start recording — Solo CLI will guide you through each episode
  2. Perform the task (pick-and-place, stacking, etc.) 10–20 times with slight variations
  3. Replay a recorded episode to check data quality
# Record teleoperation episodes (with guided prompts) solo robo --record # Skip prompts and use saved settings solo robo --record -y # Replay a recorded episode to check data quality solo robo --replay
Tip: Aim for 10–20 episodes for a basic task. Consistency matters more than volume.
Tip: Include slight variations (object positions, angles) so the model generalizes.
Tip: LeRobot GitHub (dataset format & tools): github.com/huggingface/lerobot
Checkpoint: You have 10+ recorded episodes and replay shows clean, consistent trajectories.
4
Deploy Inference Infrastructure on Nebius 10 min

Set up Nebius Serverless and Token Factory for training and deploying your VLA model.

  1. Look up your subnet ID
  2. Deploy the orchestrator on Nebius Serverless
  3. Set your Token Factory API key for GPU inference
# Get subnet export SUBNET_ID=$(nebius vpc subnet get-by-name \ --name default-subnet --format jsonpath='{.metadata.id}') # Deploy orchestrator on Serverless export AUTH_TOKEN=$(openssl rand -hex 32) nebius msp serverless v1alpha1 endpoint create \ --name openclaw-robotics \ --container-image openclaw:robotics \ --container-template-resources-platform cpu-d3 \ --container-template-resources-preset 4vcpu-16gb \ --port 8080 \ --username admin \ --password "$AUTH_TOKEN" \ --network-id <your-network-id> \ --parent-id <your-project-id> # Set Token Factory key for GPU inference export TF_API_KEY=<your-token-factory-api-key>
Checkpoint: Your serverless endpoint is active and Token Factory API key is configured.
5
Train Your VLA Model 15 min

Train a vision-language-action model on your recorded dataset. Solo CLI handles the configuration — just pick a model and go.

  1. Run the training command — Solo CLI will prompt you to choose a model
  2. Monitor the loss curve for convergence
  3. Training saves a model checkpoint you'll use for inference
# Train a VLA policy (Solo CLI handles the config) solo robo --train # Skip prompts with saved settings solo robo --train -y
Tip: ACT trains in minutes and works great for single-task policies. Paper | Code | LeRobot docs
Tip: SmolVLA is more capable but takes longer to train — good for multi-task learning. Docs
Tip: Pi0 / Pi0Fast / Pi05 — Physical Intelligence foundation models. Paper | Code | LeRobot docs
Tip: GROOT N1.5 — NVIDIA's VLA model. Paper | Code | LeRobot docs
Tip: X-VLA — cross-embodiment VLA. Paper | LeRobot docs
Checkpoint: Training completes and the loss curve shows convergence. Your model checkpoint is saved.
6
Run Autonomous Inference 10 min

Deploy your trained model and watch the arm execute the learned task autonomously — no human input required.

  1. Run the inference command with your trained model
  2. Watch the robot execute the task it learned from your demonstrations
  3. If the model doesn't generalize well, record more episodes and retrain
# Run inference with your trained model solo robo --inference # Skip prompts solo robo --inference -y
Tip: The first autonomous run is the magic moment — watch the arm do what you taught it without any human input.
Tip: If the model doesn't generalize well, record 10 more episodes with more position variety and retrain.
Checkpoint: Your robot arm executes the learned task autonomously from your trained VLA model.
7
Record Your Demo & Explore More Robots 10 min

Record a video of your robot performing autonomously, then explore other available platforms.

  1. Film a video of the full loop: teleoperation, then autonomous execution
  2. Check your Solo CLI status and saved models
  3. Save your Nebius deployment config for future use
  4. Try other robot platforms if time allows
# Check Solo CLI status and model info solo status # List downloaded models solo list # Save your Nebius deployment config nebius msp serverless v1alpha1 endpoint get $ENDPOINT_ID \ --format yaml > robotics-deploy.yaml
Tip: The best demos show the full loop: teleoperation → training → autonomous execution.
Tip: Other robots to explore: LeKiwi, OpenDroid R1D2, Unitree G1, Booster
Tip: Unitree G1 extras: SDK, Replay, IsaacLab, GROOT Wholebody
Tip: All VLA model LeRobot implementations: ACT, Pi0, Pi0Fast, Pi05, GROOT, X-VLA, SmolVLA
Checkpoint: You have a video of autonomous execution and your model + config saved for future use.
