Liberate Your OpenClaw — Run on Nebius Instead of Claude

Anthropic limits Claude model access in open agent platforms for Pro/Max subscribers. This guide gives you two alternatives using open models on Nebius: Token Factory for instant hosted inference, or Nebius AI Cloud for full control with your own GPU instances. Both use the OpenAI-compatible API — swap one URL and you're running.

Who It's For
Anyone running OpenClaw who wants fast, open-model inference without Claude API limits
Time

Token Factory — Fastest Path

1
Get Your API Key 2 min

Create a Nebius account and generate a Token Factory API key.

  1. Sign up or log in at studio.nebius.com
  2. Go to API Keys and create a new key
  3. Copy the key — you'll need it in the next step
Tip: New accounts get free credits to get started.
2
Configure OpenClaw 2 min

Point OpenClaw at Token Factory using the onboard command.

openclaw onboard --non-interactive \ --auth-choice custom-api-key \ --custom-base-url "https://api.studio.nebius.com/v1" \ --custom-model-id "meta-llama/Meta-Llama-3.1-70B-Instruct" \ --custom-api-key "<your-token-factory-api-key>" \ --secret-input-mode plaintext \ --custom-compatibility openai
Tip: Token Factory uses an OpenAI-compatible API. If your code works with OpenAI, it works with Nebius.
Checkpoint: OpenClaw prints a confirmation that the provider is configured.
3
Choose Your Model 1 min

Pick a model from the Token Factory catalog. Here are recommended options for agents:

ModelBest ForSpeed
Llama 3.1 70BGeneral agents, balanced quality/costFast
DeepSeek-V3Coding agents, technical tasksFast
Qwen 2.5 72BMultilingual, structured outputFast
Mistral LargeEuropean languages, reasoningFast
Update Your Model

To switch models, update your OpenClaw config:

{ "agents": { "defaults": { "model": { "primary": "nebius/deepseek-ai/DeepSeek-V3" } } } }
4
Test It 1 min

Verify everything works with a quick test.

# Test the connection directly curl https://api.studio.nebius.com/v1/chat/completions \ -H "Authorization: Bearer $TF_API_KEY" \ -H "Content-Type: application/json" \ -d '{ "model": "meta-llama/Meta-Llama-3.1-70B-Instruct", "messages": [{"role": "user", "content": "Hello!"}], "max_tokens": 50 }' # Or just run OpenClaw openclaw
Checkpoint: OpenClaw responds using the Nebius model. No Claude API needed.

Nebius AI Cloud — Full Control

1
Install the Nebius CLI 3 min

Install and authenticate with the Nebius CLI.

# Install Nebius CLI (macOS/Linux) curl -sSL https://storage.eu-north1.nebius.cloud/cli/install.sh | bash # Login to your account nebius auth login # Verify nebius iam whoami
Checkpoint: nebius iam whoami returns your user info.
2
Launch a GPU Instance 5 min

Spin up a GPU VM with a pre-installed inference stack. Nebius offers H100 and L40S GPUs.

  1. Choose a GPU type based on your model size
  2. Create the instance with the inference-ready image
  3. SSH in once it's running
# Create a GPU instance with 1x H100 nebius compute instance create \ --name openclaw-inference \ --platform gpu-h100 \ --preset 1gpu-128vcpu-1600gb \ --image-family ubuntu-22.04-cuda \ --disk-size 200 \ --ssh-key ~/.ssh/id_rsa.pub \ --parent-id <your-project-id> # Get the public IP nebius compute instance get-by-name \ --name openclaw-inference \ --format jsonpath='{.status.network_interfaces[0].public_ip_address.address}'
Tip: An L40S (~$0.80/hr) is enough for 70B models. Use H100 ($1.40/hr) for larger models or faster inference.
3
Start an Inference Server 5 min

Run vLLM on your GPU instance to serve an open model with an OpenAI-compatible API.

# SSH into your instance ssh ubuntu@<instance-ip> # Install vLLM pip install vllm # Start serving Llama 3.1 70B python -m vllm.entrypoints.openai.api_server \ --model meta-llama/Meta-Llama-3.1-70B-Instruct \ --port 8000 \ --tensor-parallel-size 1
Tip: vLLM gives you an OpenAI-compatible server out of the box. No custom code needed.
Checkpoint: curl http://<instance-ip>:8000/v1/models returns the model list.
4
Point OpenClaw at Your Server 2 min

Configure OpenClaw to use your self-hosted inference endpoint.

openclaw onboard --non-interactive \ --auth-choice custom-api-key \ --custom-base-url "http://<instance-ip>:8000/v1" \ --custom-model-id "meta-llama/Meta-Llama-3.1-70B-Instruct" \ --custom-api-key "nebius" \ --secret-input-mode plaintext \ --custom-compatibility openai
Checkpoint: Run openclaw and verify responses come from your self-hosted model.
5
Run Any Model You Want Ongoing

With your own GPU, you can run any model — including fine-tuned or private models not available through hosted APIs.

# Switch to DeepSeek-V3 for coding python -m vllm.entrypoints.openai.api_server \ --model deepseek-ai/DeepSeek-V3 \ --port 8000 # Or run a custom fine-tuned model python -m vllm.entrypoints.openai.api_server \ --model ./my-finetuned-model \ --port 8000
Tip: Remember to stop your instance when not in use: nebius compute instance stop openclaw-inference

  • A configured Nebius inference endpoint (hosted or self-managed)
  • The ability to switch models in seconds
  • 60–80% lower costs compared to closed APIs at high volume
