## Council notes

This is the one release every PM converged on. Risk PM and Platform PM both demanded lockdown-first and I merged their identical 'Stop the Bleed' / 'Ground Truth' work into a single co-owned v1.0 (they conceded the schema baseline must land WITH the lockdown so policies are recorded before they change). MVP PM's 'declare it shippable' framing was overruled by the reproduced breach — but I honored MVP/MVP-critique by keeping the fix afternoon-scale: drop anon SELECT/UPDATE + return=minimal + auth.uid()-bound dashboard update, NOT Platform's full submit-form Edge-Function gateway (deferred until spam actually appears, per MVP's and the Growth-critique's objection that the gateway is heavier than the breach requires and the function source is not even in this repo). Growth PM's crawlability is the one feature folded into v1.0 because it touches no policy and starts the SEO clock day one — but Growth's Node build-pipeline / per-event static pre-render was cut to a tiny commit-time sitemap script per the MVP-critique's 'this repo has no build today' objection; dynamic-event share targets wait for v1.2. rls-probe.sh is intentionally written BEFORE the lockdown change so the fix is provable, and becomes the template every later insert-only table must clone, resolving the Risk-critique that 'mirror the existing pattern' would otherwise propagate the broken policy.
